# 코드 수정 및 재배포 가이드

## 중요: 코드 변경만으로는 웹앱이 변경되지 않습니다!

### ❌ 이것만으로는 변경 안됨
```
1. 로컬에서 코드 수정
2. 파일 저장
→ 웹앱은 그대로 (변경사항 반영 안됨)
```

### ✅ 올바른 프로세스
```
1. 로컬에서 코드 수정
2. Git에 커밋
3. GitHub에 푸시
4. Vercel이 자동으로 재배포 (약 2-3분)
5. 웹앱에 변경사항 반영됨
```

---

## 자동 재배포 프로세스

### 1단계: 코드 수정

로컬에서 코드를 수정합니다:
```bash
# 예: app/page.tsx 파일 수정
# 파일 저장
```

### 2단계: Git 커밋 및 푸시

```bash
# 변경사항 추가
git add .

# 커밋
git commit -m "페이지 제목 변경"

# GitHub에 푸시
git push origin main
```

### 3단계: Vercel 자동 재배포

- ✅ **자동으로 시작됩니다!**
- GitHub에 푸시하면 Vercel이 자동으로 감지
- 새 배포가 자동으로 시작됨
- 약 2-3분 소요

### 4단계: 배포 완료 확인

1. **Vercel 대시보드 접속**
   - https://vercel.com
   - 프로젝트 선택

2. **Deployments 탭 확인**
   - 새 배포가 "Building" 상태로 표시됨
   - 완료되면 "Ready" 상태로 변경

3. **웹사이트 확인**
   - 배포 완료 후 웹사이트 새로고침
   - 변경사항이 반영되었는지 확인

---

## 재배포 과정 요약

```
코드 수정
    ↓
Git 커밋
    ↓
GitHub 푸시
    ↓
Vercel 자동 감지 (몇 초 내)
    ↓
자동 빌드 시작
    ↓
배포 진행 (약 2-3분)
    ↓
배포 완료
    ↓
웹사이트에 변경사항 반영
```

---

## 수동 재배포 (필요한 경우)

자동 재배포가 시작되지 않으면:

1. **Vercel 대시보드** → 프로젝트 선택
2. **Deployments** 탭 클릭
3. 최근 배포 옆의 "..." 메뉴 클릭
4. **"Redeploy"** 선택

---

## 배포 상태 확인

### Vercel 대시보드에서 확인

1. **Deployments** 탭
   - "Building" → 배포 진행 중
   - "Ready" → 배포 완료
   - "Error" → 오류 발생 (Build Logs 확인)

2. **Build Logs**
   - 배포 진행 상황 실시간 확인
   - 오류 발생 시 상세 내용 확인

---

## 주의사항

### ⚠️ 로컬 변경만으로는 안됨
- 로컬에서 코드를 수정하고 저장만 해도
- 웹앱은 변경되지 않습니다
- 반드시 GitHub에 푸시해야 합니다

### ⚠️ 배포 시간
- 자동 재배포는 약 2-3분 소요
- 빌드가 완료될 때까지 기다려야 합니다

### ⚠️ 배포 중
- 배포가 진행 중일 때는 이전 버전이 표시됩니다
- 배포 완료 후 새 버전이 적용됩니다

- [ ] Vercel 대시보드에서 배포 확인
- [ ] 배포 완료 후 웹사이트 확인

## 예시

### 시나리오: 페이지 제목 변경

1. **로컬에서 수정**
   ```tsx
   // app/page.tsx
   <h1>내 손안의 AI 체력 코치</h1>
   // → "학생 건강체력 관리 시스템"으로 변경
   ```

2. **Git에 커밋 및 푸시**
   ```bash
   git add app/page.tsx
   git commit -m "페이지 제목 변경"
   git push origin main
   ```

3. **Vercel 자동 재배포**
   - GitHub 푸시 후 자동으로 시작
   - 약 2-3분 대기

4. **확인**
   - 웹사이트 새로고침
   - 변경된 제목 확인
