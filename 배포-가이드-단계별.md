## 2단계: PostgreSQL 데이터베이스 생성

### 옵션 A: Neon (추천) ⭐

1. https://neon.tech 접속
2. "Sign Up" 클릭 (GitHub 계정으로 가입 가능)
3. "Create Project" 클릭
4. 프로젝트 이름 입력 (예: `paps-health`)
5. "Create Project" 클릭
6. 프로젝트 대시보드에서 **"Connection String"** 복사
   - 형식: `postgresql://user:password@host/database?sslmode=require`
   - 이 문자열을 나중에 사용합니다!

### 옵션 B: Supabase

1. https://supabase.com 접속
2. "Start your project" 클릭
3. GitHub 계정으로 가입
4. "New Project" 클릭
5. 프로젝트 정보 입력
6. Settings → Database → Connection String 복사

### 옵션 C: Vercel Postgres (가장 간단)

- Vercel에서 프로젝트 생성 후 Storage 탭에서 직접 생성 가능

## 3단계: Vercel에 배포

1. **Vercel 접속**
   - https://vercel.com 접속
   - "Sign Up" 클릭 (GitHub 계정으로 로그인)

2. **프로젝트 Import**
   - "Add New..." → "Project" 클릭
   - GitHub 저장소 목록에서 프로젝트 선택
   - "Import" 클릭

3. **프로젝트 설정**
   - Framework Preset: Next.js (자동 감지됨)
   - Root Directory: `./` (기본값)
   - Build Command: `npm run build` (자동 설정됨)
   - Output Directory: `.next` (자동 설정됨)

4. **환경 변수 설정** (중요!)
   - "Environment Variables" 섹션 클릭
   - 다음 변수 추가:
   
   ```
   이름: DATABASE_URL
   값: (2단계에서 복사한 PostgreSQL 연결 문자열)
   ```
   
   - AI 추천 기능을 사용하는 경우:
   ```
   이름: OPENAI_API_KEY
   값: (OpenAI API 키)
   ```

5. **배포 시작**
   - "Deploy" 버튼 클릭
   - 2-3분 대기

6. **배포 완료 확인**
   - 배포가 완료되면 "Visit" 버튼 클릭
   - 공개 URL 확인 (예: `https://your-app.vercel.app`)

---

## 4단계: 배포 확인 및 테스트

1. **웹사이트 접속**
   - 배포된 URL로 접속
   - 페이지가 정상적으로 로드되는지 확인

2. **기능 테스트**
   - ✅ 학생 등록 기능
   - ✅ 측정 데이터 입력
   - ✅ 결과 확인
   - ✅ 데이터 저장 확인

3. **문제 해결**
   - 오류가 발생하면 Vercel 대시보드 → Deployments → Logs 확인
   - `DATABASE_URL` 환경 변수가 올바른지 확인
