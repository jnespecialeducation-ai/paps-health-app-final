# 터미널에서 재배포하기

## 간단한 3단계

코드를 수정한 후, 터미널에서 다음 명령어만 실행하면 됩니다:

```bash
# 1. 변경사항 추가
git add .

# 2. 커밋
git commit -m "변경 내용 설명"

# 3. GitHub에 푸시 (이게 핵심!)
git push origin main
```

**끝!** 🎉

Vercel이 자동으로 재배포를 시작합니다.

---

## 전체 프로세스

```
1. 코드 수정 (에디터에서)
   ↓
2. 터미널에서 git 명령어 실행
   git add .
   git commit -m "변경 내용"
   git push origin main
   ↓
3. Vercel 자동 재배포 시작 (몇 초 내)
   ↓
4. 약 2-3분 후 배포 완료
   ↓
5. 웹사이트에 변경사항 반영됨 ✅
```

---

## 예시

### 시나리오: 버튼 색상 변경

1. **코드 수정** (에디터에서)
   ```tsx
   // app/page.tsx
   <button className="bg-blue-600">  // 파란색
   // → bg-red-600으로 변경 (빨간색)
   ```

2. **터미널에서 실행**
   ```bash
   git add .
   git commit -m "버튼 색상 빨간색으로 변경"
   git push origin main
   ```

3. **완료!**
   - Vercel이 자동으로 재배포 시작
   - 2-3분 후 웹사이트에 반영됨

---

## 주의사항

### ⚠️ 반드시 푸시해야 함
- `git add`와 `git commit`만으로는 안됨
- **`git push origin main`**이 필수!

### ⚠️ 배포 확인
- Vercel 대시보드에서 배포 상태 확인 가능
- 하지만 자동으로 진행되므로 굳이 확인 안 해도 됨

## 빠른 참고

### 변경사항이 많을 때
```bash
git add .
git commit -m "여러 파일 수정"
git push origin main
```

### 특정 파일만 변경할 때
```bash
git add app/page.tsx
git commit -m "페이지 수정"
git push origin main
```

### 커밋 메시지 예시
```bash
git commit -m "학생 목록 UI 개선"
git commit -m "측정 입력 폼 수정"
git commit -m "버그 수정: 날짜 선택 오류"
```

## 요약

**질문: 터미널에 명령어만 작성하면 되나요?**
- ✅ **네!** 정확합니다!
- `git add .` → `git commit -m "..."` → `git push origin main`
- 이 3개 명령어만 실행하면 자동으로 재배포됩니다!

**추가 작업 불필요:**
- ❌ Vercel 대시보드에서 수동 배포 클릭 불필요
- ❌ 다른 설정 변경 불필요
- ✅ Git 푸시만 하면 자동으로 처리됨
